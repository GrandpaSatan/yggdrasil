ARCHITECTURE

Network Topology

Three-node fleet: Hugin + Munin are always on (primary swarm), Morrigan is on-demand. Flows route through Odin on Munin, which dispatches to models across the fleet.

Design Principles

Cross-Architecture Review

Coder (Qwen MoE) and reviewer (Gemma dense) are different architectures — different blind spots, better bug catching.

Graceful Degradation

Flows work on Hugin+Munin alone. Morrigan is overflow for hard cases only.

LGTM Convergence

Review loops iterate until reviewer says LGTM (max 3). Quality gated in architecture.

Code Cleaner

Fine-tuned LFM2-350M normalizes messy thinking-model outputs into clean code before downstream steps.

glm-4.7-flash 🧠 Munin iGPU Reasoner / Planner "Offline Claude" — reasoning, planning, architecture. 30B MoE (3B active), 200K context, Preserved Thinking across multi-turn tool calls. #1 on 16GB-VRAM tier benchmarks, 73.8% SWE-bench. code-cleaner-350m Hugin + Morrigan Response Normalizer Fine-tuned LFM2-350M. Extracts clean code from messy thinking-model outputs. Used by benchmark smart proxy and could normalize any flow step's output. eval_loss 0.064.
all-minilm Hugin + Munin Embeddings 384-dim sentence embeddings. Used by Mimir for memory search, Muninn for code search, Huginn for indexed code retrieval. CPU-only, very fast.
odin / mimir / mcp-remote Munin Services Odin = flow engine + LLM router. Mimir = engram memory with auto-ingest. MCP-remote = HTTP MCP server (14 tools). Together these route every flow.
huginn / muninn / vision Hugin RAG / Perception Huginn = tree-sitter code indexer (Rust/Python/Go), writes to PG+Qdrant. Muninn = hybrid vector+BM25 retrieval. Vision = LFM2.5-VL on Hugin iGPU for image understanding.
llama-omni2 Hugin Voice I/O Voice server (STT + TTS + S2S). 3B model, clean 24kHz audio. Used by perceive flow when modality=voice.
