## VSCode 1.90+ Webview CSP Requirements

| Directive | Requirement |
|-----------|-------------|
| `default-src` | Must be `'none'` — no implicit fallback. |
| `script-src` | Must use per-render `nonce-<random>`. `unsafe-eval` is **banned** (enforced by VSCode 1.90 sandboxed renderer). `unsafe-inline` is **banned** for scripts. |
| `style-src` | `unsafe-inline` is **permitted** for inline styles. Extension-local CSS must be served via `webview.cspSource`. |
| `connect-src` | Must list explicit origins or schemes. `ws:` and `wss:` required for WebSocket use. |
| `img-src` | `data:` permitted for base64 images. `webview.cspSource` for local assets. |
| `font-src` | `webview.cspSource` for bundled fonts. |
| Nonce generation | Must be a fresh cryptographically random value per render, not reused. |

## Chat Panel (`chatPanel.ts`)

### Current CSP string (lines 516-523)

```
default-src 'none';
img-src ${webview.cspSource} data:;
style-src ${webview.cspSource} 'unsafe-inline';
font-src ${webview.cspSource};
script-src 'nonce-${nonce}';
connect-src ws: wss: http: https:
```

Resolved at runtime (example):

```
default-src 'none';
img-src vscode-webview://abc123 data:;
style-src vscode-webview://abc123 'unsafe-inline';
font-src vscode-webview://abc123;
script-src 'nonce-abc...xyz';
connect-src ws: wss: http: https:
```

### Compliance check

| Directive | Status | Notes |
|-----------|--------|-------|
| `default-src 'none'` | PASS | Correct — no implicit fallback. |
| `img-src ${cspSource} data:` | PASS | Extension assets + base64 thumbnails allowed. |
| `style-src ${cspSource} 'unsafe-inline'` | PASS | `unsafe-inline` is permitted for styles in VSCode 1.90+. Retro-theme CSS is injected via `<link>` from cspSource, and dynamic theme class toggling uses inline style attributes — both covered. |
| `font-src ${cspSource}` | PASS | VT323 and JetBrains Mono fonts are bundled in `media/fonts/` and served via cspSource. |
| `script-src 'nonce-${nonce}'` | PASS | No `unsafe-eval`. No `unsafe-inline`. All `<script>` tags carry the nonce (verified at lines 544-556, 614-615). |
| `connect-src ws: wss: http: https:` | PASS | Required for: Odin HTTP (`http:`), streaming SSE (`http:`), voice WebSocket (`ws:` and `wss:`). Sprint 062 added `ws: wss:` — still present and correct. |
| Nonce freshness | PASS | `getNonce()` at line 622 generates a new random value per render using `crypto.randomBytes`-equivalent. Not reused across renders. |
| `unsafe-eval` | PASS (absent) | Not present. Prism.js syntax highlighter is loaded as static files via nonce — does not require eval. |

### Violations found

None. Chat panel CSP is fully compliant with VSCode 1.90+ requirements.

## Sprint 062 `connect-src` Verification

Sprint 062 added voice WebSocket support. The voice client (`voice-client.js`) opens a
`WebSocket` connection from within the chat panel webview.

- `connect-src ws: wss: http: https:` is present in `chatPanel.ts` CSP (line 522).
- `ws:` covers `ws://10.0.65.8:8080/v1/voice` (Odin Munin, non-TLS).
- `wss:` covers future TLS deployments.
- The voice client script is conditionally included only when `voiceEnabled` is true
  (line 615): `${voiceEnabled ? '<script nonce="${nonce}" src="${voiceClientUri}"></script>' : ""}`.
  The nonce is applied correctly.

Status: COMPLIANT. No regressions from Sprint 062.

## Recommendations

1. **Settings panel `connect-src`:** If `settings.js` is ever extended to make direct
   `fetch()` calls (e.g., for health probes bypassing the extension host), add
   `connect-src http: https:` to the settings panel CSP. Currently not needed.

2. **`connect-src` specificity:** The chat panel uses `connect-src ws: wss: http: https:`.
   For a tighter posture, restrict to `connect-src ${webview.cspSource} ws://10.0.65.8:8080
   wss://10.0.65.8:8080 http://10.0.65.8:8080 http://10.0.65.8:9090`. The current broad
   scheme-based rule is acceptable for a homelab context where origins are controlled.

3. **`worker-src`:** If Web Workers are ever added (e.g., for WASM-based audio processing),
   a `worker-src 'none'` or `worker-src ${cspSource}` directive will be required.
   Currently no workers are used — `worker-src` falls back to `default-src 'none'` which
   blocks workers by default (correct behavior).

## Verification Procedure

To confirm zero CSP violations during a live voice session:

1. Install the VSIX in VS Code.
2. Open the Chat panel (`Ctrl+Shift+I`).
3. Open DevTools: `Help → Toggle Developer Tools` → Console tab.
4. Set `yggdrasil.voice.enabled = true` in settings.
5. Send a chat message and observe the stream.
6. Click the mic button, speak a phrase, release.
7. Observe Console: **zero lines** beginning with `Content Security Policy` or `Refused to`.

If any CSP violation appears, cross-reference the blocked directive against the current
CSP strings above and add the required source expression.
